How Two-Factor Authentication Works

Understanding the security behind TOTP codes

üîë

Secret Key

When you enable 2FA, the service gives you a secret key (usually shown as a QR code). This key is shared between you and the service.

‚è∞

Time-Based

TOTP uses the current time combined with your secret key to generate a unique 6-digit code that changes every 30 seconds.

üîê

HMAC-SHA1

The algorithm uses HMAC-SHA1 to hash the time counter with your secret, then extracts 6 digits from the result.

‚úÖ

Verification

The service generates the same code using the shared secret. If your code matches, you're authenticated!

API Documentation

Integrate Miro into your applications with our simple REST API

POST /api?action=getcode

Generate a 2FA code from a secret key

cURL Example
curl -X POST "https://qr-2fa.vercel.app/api?action=getcode" \
  -H "Content-Type: application/json" \
  -d '{"secret": "JBSWY3DPEHPK3PXP"}'
Response
{
  "secret": "JBSWY3DPEHPK3PXP",
  "otp": "123456"
}
POST /api?action=qrcode

Decode QR code image and generate 2FA code

cURL Example
curl -X POST "https://qr-2fa.vercel.app/api?action=qrcode" \
  -H "Content-Type: application/json" \
  -d '{"image": "data:image/png;base64,..."}'
Response
{
  "secret": "JBSWY3DPEHPK3PXP",
  "otp": "123456"
}
POST /api?action=parse

Parse an otpauth:// URL and generate code

cURL Example
curl -X POST "https://qr-2fa.vercel.app/api?action=parse" \
  -H "Content-Type: application/json" \
  -d '{"url": "otpauth://totp/Google:user@gmail.com?secret=JBSWY3DPEHPK3PXP"}'
Response
{
  "secret": "JBSWY3DPEHPK3PXP",
  "otp": "123456"
}
